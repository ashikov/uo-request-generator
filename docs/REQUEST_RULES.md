# Правила составления заявок

## Содержание и тон

- Использовать спокойный человеческий тон без канцелярита
- Описывать одну наблюдаемую проблему компактно, но без потери известных
  подробностей
- Сохранять переданные объект, место, признаки, длительность и повторяемость
- Отделять известное практическое значение проблемы от её описания
- Формулировать от одного до трёх конкретных выполнимых требований
- Преобразовывать эмоции в наблюдаемые факты без домысливания
- Формировать итоговый раздел «Прошу:» как нумерованный список требований
- Укладывать полный текст с требованиями в установленный лимит результата

## Ограничения

- Не придумывать место, причины, виновников, повреждения, риски, последствия и
  выполненные работы
- Не добавлять неподтверждённые обвинения
- Не добавлять законодательство самостоятельно
- Не создавать отдельный раздел «Правовое обоснование»
- Не просить предоставить письменный ответ по умолчанию
- Не устанавливать конкретный срок исполнения по умолчанию
- Не создавать впечатление, что приложение самостоятельно отправляет заявку

## Короткие примеры

**Эмоция:** «В подъезде опять невозможно находиться»

**Наблюдаемый факт:** «В подъезде сохраняется резкий неприятный запах»

**Размытая просьба:** «Сделайте уже что-нибудь»

**Выполнимая просьба:** «Прошу проверить источник запаха и устранить причину»

**Неподтверждённое объяснение:** «Трубу точно повредили рабочие»

**Безопасная формулировка:** «На трубе заметна течь. Прошу провести осмотр и
устранить неисправность»
