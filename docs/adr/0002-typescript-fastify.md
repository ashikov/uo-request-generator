# ADR-0002: TypeScript, Fastify и статический frontend

## Статус

Accepted

## Контекст

Приложение состоит из одной минималистичной веб-формы. Владелец проекта знаком с
JavaScript и должен самостоятельно читать и ревьюить код. Текущему интерфейсу
не требуется компонентный frontend-фреймворк. Важны прозрачный поток выполнения
и небольшое количество зависимостей.

## Решение

Использовать TypeScript для серверного и доменного кода, Fastify как
HTTP-сервер, обычные HTML и CSS, а также минимальный browser JavaScript.
Доменную логику сохранять в `packages/core`, а интеграцию с генерацией отделять
провайдер-независимой LLM-границей.

## Последствия

Положительные последствия:

- Прямой и понятный поток запроса
- Небольшой объём фреймворк-специфичных знаний
- Простой код для ревью
- Небольшое количество runtime-зависимостей
- Возможность позднее заменить frontend без изменения доменного ядра

Ограничения:

- При подтверждённом усложнении интерфейса может понадобиться отдельный
  frontend-фреймворк
- Повторно используемые UI-компоненты пока поддерживаются вручную
