# ADR-0001: Монорепозиторий и модульный монолит

## Статус

Accepted

## Контекст

Первая версия обслуживает один короткий веб-сценарий для одного дома. Отдельные
сервисы увеличили бы стоимость разработки и эксплуатации без практической
пользы.

## Решение

Хранить веб-приложение и внутренние пакеты в pnpm-монорепозитории. Запускать
систему как один Fastify-процесс в одном контейнере, сохраняя явные границы
между `web`, `core` и `llm`.

## Последствия

Разработка, проверки и поставка остаются простыми. Пакеты можно изменять
согласованно. Разделение на сервисы потребует нового решения и измеримой причины.
